# 故事书
1. 用户在首页进行导航，进入校友总会页面，校友会分布页面，学院校友会页面，杰出校友页面，校友图册页面
2. 在五大基础页面中，会有一个返回首页的按钮，点击后回到导航首页
3. 用户在校友总会页面会看到校友总会介绍
4. 用户在校友分会页面会看到各个地区校友会的分布
    1. 用户需要校友会的国内分布情况
    2. 用户需要校友会的国际分布情况
    3. 用户需要校友会的行业分布情况，需要跟甲方进一步确认
    4. 用户可以通过底部按钮，对该页面进行实时切换
    5. 在校友分布页面，点击具体某一个校友分会会进入校友分会介绍
    6. 当进入子页面时，返回首页按钮变成返回上一页的按钮，点击返回行一页，所有的子页面都是这个逻辑
5. 用户在学院校友会页面，会看到各个学院校友会的卡片，点击进入详细页面
6. 杰出校友，已基本实现
7. 用户在校友图册页面看到每个年级的所有毕业照片
    1. 用户在底部时光轴点击具体的时间节点，该节点会跳转到时光轴的中间
    2. 用户在底部时光桌点击具体的时间节点，中间会轮播该节点的毕业照片
    3. 用户点击毕业照片后会进入类似于学院校友分会的界面，罗列出所有毕业照片
    4. 点击毕业照片进入大屏显示，再点击一下退出

## 校友图册页面出现卡顿，如何定位到到底什么才是导致卡顿的真实原因
1. 使用浏览器性能分析工具进行分析
- `commit`：提交渲染树。浏览器主线程完成布局和样式计算，将数据提交给合成线程显示。86.4ms(20.6%)。
原因：拖动时大量改变DOM，DOM树的高度太大
- `pre-paint`：浏览器在实际绘制像素前，检查哪些元素发生改变，哪些需要重绘。57.6ms(13.7)。
原因：卡片的阴影，圆角等要素。
- `Evaluate script`：self time 3.3ms，total time 228.5ms。js代码执行快，但是它触发了后续及其耗时的渲染链条。

2. 图片的像素可能有1920px宽，卡片显示宽度可能只有300px宽，浪费GPU性能。

## 如何解决这个问题
1. 使用缩略图：在前端罗列列表使用生成的小图，当点击显示的时候显示大图。
